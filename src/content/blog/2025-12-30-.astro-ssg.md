在搭建个人技术博客时，我一开始最困惑的问题是：  
**为什么 Astro 这么强调 SSG？**
在真正用过之后，我才意识到：  
SSG 并不是“高级”，而是**非常克制、非常工程化的选择**。



## WHAT: 什么是SSG？

**SSG（Static Site Generation，静态站点生成）**，指的是：

> 在构建阶段就把页面生成好，而不是等用户访问时再生成。

对于博客来说，这意味着：

- 页面在上线前已经是完整 HTML
- 用户访问时不需要任何额外计算
- 打开速度极快，SEO 友好



## HOW: Astro是如何实现SSG的？

Astro 的核心思路是：

> **默认不向浏览器发送 JavaScript**

在构建阶段，Astro 会：

1. 读取 Markdown 内容
2. 根据路由规则生成 HTML 文件
3. 对代码块进行高亮（Shiki）
4. 输出纯静态文件

最终部署到服务器的，只是一些 `.html`、`.css` 和少量资源文件。



## WHY: 为什么博客非常适合SSG？

从博客的特性来看：

- 内容更新频率低
- 几乎没有实时交互
- 访问主要来自搜索引擎

这和 SSG 的优势是完全匹配的。

| 需求 | SSG 表现 |
|----|----|
| 打开速度 | 极快 |
| SEO | 非常好 |
| 成本 | 几乎为 0 |
| 稳定性 | 极高 |
